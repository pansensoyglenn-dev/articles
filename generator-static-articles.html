Static Article Generator

Poetic Codes — Fixes Google AdSense "thin content" by generating static HTML files for every article in Firebase

Why This Tool Exists

The Problem: Your articles live at URLs like article.html?id=abc123. Google's crawler visits that URL and sees a blank page — the content only appears after JavaScript fetches it from Firebase. Google cannot read it, so it calls every article "thin content."
1
This tool connects to your Firebase, reads all your articles, and generates a proper static .html file for each one with the full content baked directly into the HTML.
2
Each file is named after the article's slug (e.g. why-clean-code-matters.html) — a real URL that Google can fully crawl and index.
3
Upload all generated files to your Cloudflare Pages repo alongside your existing files. Each article becomes a proper standalone page.
4
Submit a sitemap.xml (also generated here) to Google Search Console listing all article URLs. Google will crawl and index them.

Step 2 — Generate Static HTML Files

This will create one .html file per article with the full content baked in, plus a sitemap.xml. Each file downloads automatically.

Step 3 — Deploy to Cloudflare Pages

1
All files were downloaded to your Downloads folder. Create a folder called poetic-articles and move them all there.
2
Go to your GitHub repo (pansensoyglenn-dev/articles or wherever your Cloudflare Pages deploys from) and upload all the .html files plus the sitemap.xml to the root.
3
Cloudflare Pages will auto-deploy. Each article will now be live at its own URL: https://code-avs.pages.dev/why-clean-code-matters.html
4
Go to Google Search Console → Sitemaps → Submit https://code-avs.pages.dev/sitemap.xml. Google will discover and crawl every article.
5
In Google AdSense → Policy Center → click Request Review. With 50 fully-crawlable articles, the thin content rejection will be resolved.
