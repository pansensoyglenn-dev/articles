Accessibility at Poetic Codes

I believe that thoughtful writing should be available to everyone, regardless of ability. This page outlines my commitment to inclusive design and the accessibility features we provide.

My Commitment

Poetic Codes is dedicated to ensuring that my website and the essays I publish are accessible to people with disabilities. We strive to meet or exceed the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards. We continuously test and improve our site to provide a seamless experience for all readers.

Accessibility Features

Keyboard Navigation

Every interactive element on this site can be accessed using a keyboard alone. Use the Tab key to move forward, Shift+Tab to move backward, and Enter to activate buttons and links. The hamburger menu, filter buttons, and essay cards all respond to keyboard input.

Screen Reader Support

My HTML is semantically structured with proper headings, landmarks, and ARIA labels where needed. All images include descriptive alt text, and dynamic content (like the essay grid) is announced correctly. I recommend using recent versions of NVDA, JAWS, or VoiceOver for the best experience.

Color Contrast & Readability

I maintain a high‑contrast color scheme that meets WCAG guidelines. Text is set in a legible serif font (Crimson Pro) with adjustable line height. The background is a dark obsidian, reducing eye strain during long reading sessions.

Responsive Design & Text Scaling

The layout adapts to any screen size, from mobile phones to large desktop monitors. You can resize text up to 200% without breaking the layout or losing functionality. Pinch‑to‑zoom is enabled on touch devices.

Clear Navigation & Structure

All pages have consistent headers, breadcrumbs, and a table of contents on article pages. The search and filter functions allow you to quickly locate content without scrolling excessively.

Assistive Technology Testing

I regularly test our site with the following tools and technologies:

  • NVDA (Windows)
  • VoiceOver (macOS/iOS)
  • TalkBack (Android)
  • ChromeVox (Chrome)
  • WAVE Web Accessibility Evaluation Tool
  • axe DevTools

Ongoing Improvements

Accessibility is not a one‑time task; it is an ongoing process. As we add new features and essays, we ensure that accessibility remains a priority. If you encounter any barriers while using our site, please let me know.

Feedback & Assistance

Your feedback helps me improve. If you have difficulty accessing any part of Poetic Codes, or if you have suggestions for enhancements, please reach out using my contact form or email us directly at pansensoyglenn@gmail.com. We will respond promptly and work to resolve any accessibility issues.
